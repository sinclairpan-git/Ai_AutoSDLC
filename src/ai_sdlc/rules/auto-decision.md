# 自主决策规则

> 本规则在遇到歧义、NEEDS CLARIFICATION 或需要做选择时激活。

## 决策分层

遇到任何需要做选择的场景时，按以下优先级逐层查找答案：

```
Layer 1: 查预决策矩阵（.ai-sdlc/profiles/decisions.yml）
  ↓ 无匹配
Layer 2: 查宪章约束（.ai-sdlc/memory/constitution.md）
  ↓ 宪章未覆盖
Layer 3: 查 PRD 原文（寻找隐含答案）
  ↓ PRD 未覆盖
Layer 4: AI 自主判断（受约束）
  ↓ 命中升级条件
Layer 5: 阻断等待人工
```

## Layer 4：AI 自主判断的约束

当走到 AI 自主判断时，必须遵循以下约束：

1. **最简原则**：在满足需求的多个方案中，选择最简单的
2. **最安全原则**：在安全性有差异的多个方案中，选择最安全的
3. **MVP 原则**：只选择当期 MVP 必需的方案，不为未来预埋
4. **可逆原则**：优先选择后续可以修改的方案，避免不可逆决策
5. **一致原则**：与项目中已有的模式保持一致

## Layer 5：必须阻断的场景

以下情况不允许 AI 自主决策，必须阻断流水线：

- 涉及用户隐私或数据合规
- 涉及资金、支付或财务逻辑
- PRD 中存在自相矛盾的需求（A 说要、B 说不要）
- 影响范围无法评估（不知道改了会影响什么）
- 需要与外部系统的真实联调才能确认的问题
- 宪章明确禁止的操作

## 决策记录

每个自动决策都必须记录以下信息：

```markdown
#### 自动决策 AD-NNN
- **问题**：[原始歧义或 NEEDS CLARIFICATION 文本]
- **决策**：[选择的答案]
- **来源**：decisions.yml / constitution / prd / ai_judgment
- **理由**：[为什么选这个]
- **替代方案**：[考虑过但没选的方案]
- **风险**：[如果选错了可能的后果]
- **可逆性**：高 / 中 / 低
```

## 累计限制

当 AI 自主判断（Layer 4）累计达到一定次数时触发暂停：

- 单个阶段内 AI 自主判断 > 5 次 → 在当前阶段结束时输出所有决策供审查
- 整条流水线内 AI 自主判断 > 15 次 → 暂停流水线，输出完整决策日志
- 任何单个 AI 自主判断的影响范围涉及 > 3 个文件 → 该决策需要单独标注为"高影响决策"

## 与 decisions.yml 的协作

decisions.yml 是项目级的预配置答案库。当 AI 在 Layer 4 做了一个高质量的自主判断后，可以在 task-execution-log.md 中建议将该判断追加到 decisions.yml，以便后续 feature 复用。
