# AI-SDLC v0.8.4 Release Notes

## Positioning

`0.8.4` is the Vue3 public PrimeVue default-provider governance release. It
changes the ordinary frontend recommendation for new, unspecified frontend
requirements to `vue3` + `public-primevue` + `modern-saas`, while preserving the
explicit `vue2` + `enterprise-vue2` path for built-in enterprise component
library users.

## What Changed

### 1. Vue3 public PrimeVue is the ordinary default provider

- Default solution confirmation now recommends `frontend_stack=vue3`,
  `provider_id=public-primevue`, and `style_pack_id=modern-saas`.
- `enterprise_provider_eligible=True` no longer changes the ordinary default by
  itself.
- Explicit enterprise Vue2 requests still preserve `vue2` / `enterprise-vue2`
  and their enterprise preflight diagnostics.

### 2. Managed Vue3 frontend template is runnable

- The generated managed frontend includes Vite, Vue3, PrimeVue theme
  initialization, UnoCSS, CSS variables, Pinia, Vue Router, and Base component
  wrappers.
- Provider core packages remain `primevue` and `@primeuix/themes`; template
  runtime/dev dependencies are tracked separately.
- Direct `primevue/*` imports from business/view source roots are reported as
  non-blocking import-boundary warnings.

### 3. Browser quality gate has blocker and advisory layers

- Generated Vue3 public-primevue browser gates start the Vite dev server and run
  Web smoke against the rendered app.
- Blank render, fatal browser console errors, and page errors are blockers.
- Desktop `1440x900` and mobile `390x844` screenshots are captured as visual
  evidence with provider, style pack, delivery entry, timestamp, viewport, and
  smoke result metadata.
- Visual structure, horizontal overflow, Button/Input/Select/Dialog/Form
  interaction coverage, Dialog close/focus-return risks, missing labels/ARIA
  names, and keyboard focus visibility issues are first-version advisory
  warnings.

## Compatibility Notes

- Existing projects that explicitly select `enterprise-vue2` are not migrated to
  Vue3.
- Projects that rely on the ordinary unqualified frontend default should expect
  new generated frontend output to be Vue3 public-primevue based.
- Pixel diff and full WCAG-style audits are not blockers in this release; the
  evidence format is prepared for future stricter gates.

## Install And Upgrade

Use the platform release asset from
`https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/download/v0.8.4`.

- Windows: download the `.zip` asset, for example
  `ai-sdlc-offline-0.8.4-windows-amd64.zip`, then run
  `install_offline.ps1 -AddToPath` for new installs or `--upgrade-existing` /
  `-UpgradeExisting` when replacing an older CLI.
- macOS / Linux: download the matching `.tar.gz` asset, unpack it, then run
  `install_offline.sh --add-to-path` for new installs or
  `--upgrade-existing` when replacing an older CLI.
- If an older installed CLI reports `No such command 'install'`, replace it
  once with the matching platform asset and `--upgrade-existing`; after that,
  use `ai-sdlc self-update check` for future releases.
- If PATH has not been refreshed yet, use the bundle Python entry directly with
  `python -m ai_sdlc`.

Release assets are produced by `release-build.yml`. After the GitHub Release is
published and assets are attached, `release-artifact-smoke.yml` verifies the
published `.zip` and `.tar.gz` assets from the release page.
