# AI-SDLC v0.8.1 Release Notes

## Positioning

`0.8.1` is a Windows upgrade reliability patch. It keeps the user-facing
install and upgrade commands unchanged while making the Windows offline package
usable for existing Python 3.11 and Python 3.12 AI-SDLC runtimes.

## What Changed

### 1. Windows packages include Python 3.11 and 3.12 dependency wheels

- The Windows release bundle now records `supported_python_versions` in
  `bundle-manifest.json`.
- The Windows release build downloads dependency wheels for both Python 3.11 and
  Python 3.12 into the same `ai-sdlc-offline-0.8.1-windows-amd64.zip` package.
- Users do not need to choose a Python version; `pip` selects the matching wheel
  for the runtime behind the current `ai-sdlc` command.

### 2. Upgrade rescue keeps the same command

- Existing users can still run `ai-sdlc self-update check` when that command is
  available.
- If an older runtime cannot self-update cleanly, the rescue command remains
  `powershell -ExecutionPolicy Bypass -File .\install_offline.ps1 -UpgradeExisting`
  from the extracted Windows bundle directory.
- The installer validates the current runtime against the supported Python
  version list instead of rejecting every runtime that does not match a single
  build-time Python ABI.

### 3. POSIX installers understand the same manifest shape

- `install_offline.sh` accepts both the old `wheel_python_version` manifest field
  and the new `supported_python_versions` list.
- Existing macOS and Linux package behavior remains unchanged.

## Compatibility Notes

- Windows x64 uses `ai-sdlc-offline-0.8.1-windows-amd64.zip`.
- macOS / Linux use `.tar.gz` bundles:
  `ai-sdlc-offline-0.8.1-macos-arm64.tar.gz` and
  `ai-sdlc-offline-0.8.1-linux-amd64.tar.gz`.
- Existing `v0.7.x` users can upgrade directly to `v0.8.1` with the matching
  platform package or `ai-sdlc self-update check`.
- Users stuck on older releases, including CLIs that report
  `No such command 'install'`, should download the matching package from
  `releases/download/v0.8.1` and run `--upgrade-existing` once.
- Users who want direct `ai-sdlc ...` commands should install with
  `-AddToPath` on Windows or `--add-to-path` on macOS / Linux.
- Users who cannot rely on `PATH` can keep using `python -m ai_sdlc ...` from
  the installed runtime.

## Verification

Required before marking `v0.8.1` complete:

- `uv run pytest`
- `uv run ruff check src tests`
- `uv run ai-sdlc verify constraints`
- GitHub PR checks for this release branch
- Codex review with no unresolved blocking findings
- Windows cloud E2E for `v0.7.5 -> v0.8.1` rescue upgrade on Python 3.11 and
  Python 3.12
- Windows cloud E2E for `v0.7.6 -> v0.8.1` self-update or rescue upgrade on
  Python 3.11 and Python 3.12
- `release-build.yml` for tag `v0.8.1`
- `release-artifact-smoke.yml` for tag `v0.8.1`
