# AI-SDLC v0.8.5 Release Notes

## Positioning

`0.8.5` is a post-`0.8.4` user-experience patch. It keeps the Vue3 public
PrimeVue default-provider behavior from `0.8.4`, and focuses on making upgrade
and initialization guidance understandable when adapter host-ingress evidence is
not machine-verifiable from a regular terminal.

## What Changed

### 1. Adapter verification wording no longer looks like an upgrade failure

- Default CLI output now explains that `materialized` / `unverified` adapter
  states usually mean the project rules were installed, but the current terminal
  cannot machine-prove that the AI chat host loaded them automatically.
- The beginner-facing result explicitly says this is not an install or upgrade
  failure.
- Raw adapter truth remains available through `ai-sdlc adapter status --json`
  for troubleshooting.

### 2. `run --dry-run` gives a plain next action

- Dry-run continues in the materialized/unverified adapter state.
- If open gates remain, the output now explains that this normally means the
  requirement or tests are unfinished, not that the upgrade failed.
- The normal next step points back to Codex/AI chat and the project rules,
  instead of pushing users into repeated diagnostic commands.

### 3. User guide documents the confusing upgrade messages

- The Chinese user guide now explains `adapter ingress`,
  `materialized (unverified)`, and `host ingress` in ordinary user terms.
- The common-error table now tells users when to ignore the diagnostic wording
  and when to run `adapter status` / `adapter status --json`.

## Compatibility Notes

- No provider, scaffold, workflow, or frontend governance contract changes are
  introduced in this patch.
- Existing projects that explicitly selected `enterprise-vue2` or already use
  the `0.8.4` Vue3 public PrimeVue default continue to behave the same.
- Machine-readable diagnostics keep the same state names for framework and CI
  consumers.

## Install And Upgrade

Use the platform release asset from
`https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/download/v0.8.5`.

- Windows: download the `.zip` asset, for example
  `ai-sdlc-offline-0.8.5-windows-amd64.zip`, then run
  `install_offline.ps1 -AddToPath` for new installs or
  `install_offline.ps1 -UpgradeExisting` when replacing an older CLI.
- macOS / Linux: download the matching `.tar.gz` asset, unpack it, then run
  `install_offline.sh --add-to-path` for new installs or
  `--upgrade-existing` when replacing an older CLI.
- If an older installed CLI reports `No such command 'install'`, replace it
  once with the matching platform asset and `--upgrade-existing`; after that,
  use `ai-sdlc self-update check` for future releases.
- If PATH has not been refreshed yet, use the bundle Python entry directly with
  `python -m ai_sdlc`.

Release assets are produced by `release-build.yml`. After the GitHub Release is
published and assets are attached, `release-artifact-smoke.yml` verifies the
published `.zip` and `.tar.gz` assets from the release page.
