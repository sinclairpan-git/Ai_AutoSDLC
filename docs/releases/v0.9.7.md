# AI-SDLC v0.9.7 Release Notes

## Positioning

`0.9.7` is the consumer-safety and lifecycle-truth release. It prevents
AI-SDLC's own repository-development constraints from leaking into ordinary
consumer projects, while keeping portable project checks active. It also
consolidates the read-only, resume, merge, and close paths that accumulated
after `0.9.6`.

This release does not restart the completed code-reduction programme. The
bounded deduplication work that proved safe is included, remaining structural
debt stays non-blocking, and normal feature development remains the primary
delivery path.

## What Changed

### 1. Framework-only constraints are isolated from consumer projects

- Repository identity now requires both the `ai-sdlc` package metadata and the
  framework source package before framework-internal release, backlog, adapter,
  profile, and feature-contract checks can run.
- Ordinary projects retain portable constitution, task acceptance, comment,
  text-quality, branch-lifecycle, and relevant frontend import checks.
- Inconsistent partial framework identity fails closed instead of silently
  selecting the wrong constraint set.
- Consumer `014` work items no longer receive framework-only frontend runtime
  attachment output, and empty frontend scans no longer manufacture a
  PrimeVue-specific report.

### 2. Read-only commands no longer mutate adapter state

- Work-item truth and close checks defer adapter reconciliation until a clean,
  explicitly writable path is selected.
- Program read paths separate repository truth from adapter runtime state.
- Fresh-main tests guard against accidental writes to checked-out source and
  user projects.

### 3. Lifecycle and recovery truth is more reliable

- Program status honors `close_pending` instead of reporting premature
  closure.
- Program snapshots remain stable across pull-request squash merges.
- Resume packs support portable, lossless reconstruction and retain active
  work-item linkage.
- Source inventory, closure receipts, and fresh-main acceptance now converge on
  the same merged revision.

### 4. Bounded maintenance reduced duplicate implementation

- Shared text, mapping, model-string, frontend artifact-path, and frontend
  artifact-loader logic use reviewed private helpers instead of parallel
  implementations.
- The reduction route is closed after its bounded terminal work item; no new
  reduction work item is created by this release.

## Compatibility Notes

- The Vue3 v1.8 `public-primevue` defaults from `0.9.6` remain unchanged.
- Explicit `vue2` / `enterprise-vue2` compatibility remains available.
- `ai-sdlc init .` remains the normal setup and managed-rule refresh entry.
  Recognized framework-managed adapter templates may be refreshed; files with
  user modifications are preserved and reported as skipped.
- `adapter status` and `run --dry-run` remain troubleshooting commands, not
  mandatory beginner-path steps.
- `python -m ai_sdlc ...` remains the module-entry fallback when the shell has
  not refreshed PATH.

## Install And Upgrade

Use the platform release asset from
`https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/download/v0.9.7`.

- Windows: download `ai-sdlc-offline-0.9.7-windows-amd64.zip`, then run
  `install_offline.ps1 -AddToPath` for new installs or
  `install_offline.ps1 -UpgradeExisting` when replacing an older CLI.
- macOS / Linux: download the matching `.tar.gz` asset, unpack it, then run
  `install_offline.sh --add-to-path` for new installs or
  `--upgrade-existing` when replacing an older CLI.
- Existing users can run `ai-sdlc self-update install --version 0.9.7` after
  the release assets are published.
- Existing users may run `ai-sdlc self-update check` before upgrading. If an
  older CLI reports `No such command 'install'`, use the matching platform
  package and its upgrade-existing installer path instead of retrying the
  unsupported command.
