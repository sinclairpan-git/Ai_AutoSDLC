# AI-SDLC v0.8.8 Release Notes

## Positioning

`0.8.8` is a user-upgrade prompt patch for `0.7.6+` and `0.8.x` installs.
It makes newer-release discovery visible in both terminal use and AI-assisted
sessions, while keeping redirected output and automation paths safe.

## What Changed

### 1. CLI commands now prompt before upgrading when a newer release exists

- When an installed `ai-sdlc` runtime detects a newer GitHub Release, ordinary
  interactive terminal commands now ask:
  `当前AI-SDLC版本是XXX，最新版本是XXX，是否升级？回复 y/n`
- Answering `y` starts the existing automatic self-update flow.
- Answering `n` skips only the current invocation and lets the original command
  continue.
- The latest-release check still uses the existing daily cache policy.

### 2. AI and non-interactive sessions surface the same update decision

- When `ai-sdlc` is run by Codex, VS Code, AI plugins, or other non-interactive
  shells, the command output now includes the same user-facing update question.
- The output tells the AI assistant to run `ai-sdlc self-update check` only after
  the user confirms the update in chat.
- This keeps update decisions visible to the user instead of silently hiding
  them inside framework diagnostics.

### 3. Redirected stdout stays safe

- Interactive upgrade confirmation only runs when `stdin`, `stdout`, and
  `stderr` are all terminals.
- Commands such as `ai-sdlc status > status.txt` no longer risk writing updater
  panels into redirected command output.
- Non-interactive and redirected paths use the non-blocking notice surface
  instead of waiting for terminal input.

## Compatibility Notes

- No frontend provider, scaffold, Vue3 PrimeVue, Vue2 enterprise, or managed
  delivery contract changes are introduced in this patch.
- The automatic updater, unknown-channel upgrade handling, PATH repair, and
  beginner-safe `init .` output from `0.8.7` remain unchanged.
- Machine-readable diagnostic fields keep their existing internal names for
  framework and CI consumers.

## Install And Upgrade

Use the platform release asset from
`https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/download/v0.8.8`.

- Windows: download `ai-sdlc-offline-0.8.8-windows-amd64.zip`, then run
  `install_offline.ps1 -AddToPath` for new installs or
  `install_offline.ps1 -UpgradeExisting` when replacing an older CLI.
- macOS / Linux: download the matching `.tar.gz` asset, unpack it, then run
  `install_offline.sh --add-to-path` for new installs or
  `--upgrade-existing` when replacing an older CLI.
- Existing `0.7.6+` users can normally run `ai-sdlc self-update check` to get
  this release directly.
- If `ai-sdlc` is not on PATH yet, run the same check through
  `python -m ai_sdlc self-update check`.
- If an older installed CLI reports `No such command 'install'`, replace it
  once with the matching platform asset and `--upgrade-existing`; after that,
  use `ai-sdlc self-update check` for future releases.

Release assets are produced by `release-build.yml`. After the GitHub Release is
published and assets are attached, `release-artifact-smoke.yml` verifies the
published `.zip` and `.tar.gz` assets from the release page.
