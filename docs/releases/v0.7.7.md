# AI-SDLC v0.7.7 Release Notes

## Positioning

`0.7.7` is a legacy-upgrade correction on top of `0.7.6`. It keeps the new normal update path as one command:

```bash
ai-sdlc self-update check
```

It also fixes the old-version rescue path. Users on older CLIs may not have the newer self-update subcommands, so the published guidance now uses the platform install package instead of asking users to run Python or pip commands.

## What Changed

### 1. Install packages can upgrade the current old CLI entry

macOS / Linux and Windows install packages now expose the same legacy upgrade concept.

The offline install package now supports an explicit upgrade mode:

```bash
./install_offline.sh --upgrade-existing
```

Windows PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -File .\install_offline.ps1 -UpgradeExisting
```

This mode locates the `ai-sdlc` command currently resolved by `PATH`, installs the bundled wheel into that command's runtime, and verifies that the upgraded command exposes the newer self-update surface.

### 2. Legacy docs no longer expose pip rescue commands

Older users may see `No such command 'install'` or vague channel guidance from pre-`0.7.6` CLIs. The documented path is now:

- download the matching release asset from `releases/download/v0.7.7`
- extract the package
- run `--upgrade-existing`
- then use `ai-sdlc self-update check` for future updates

### 3. `ai-sdlc --version` is now available

The CLI now supports `ai-sdlc --version` so install and upgrade flows can give users a simple visible verification command.

## Compatibility Notes

- macOS Apple Silicon uses `ai-sdlc-offline-0.7.7-macos-arm64.tar.gz`
- Linux x64 uses `ai-sdlc-offline-0.7.7-linux-amd64.tar.gz`
- Windows x64 uses `ai-sdlc-offline-0.7.7-windows-amd64.zip`
- Source checkouts and editable installs remain development environments; they should not be silently overwritten by release installers
- If `--upgrade-existing` cannot safely identify the old command's runtime, it fails closed instead of pretending the upgrade succeeded
