# AI-SDLC v0.7.13 Release Notes

## Positioning

`0.7.13` is a Windows installer hotfix on top of `0.7.12`.

It fixes a Windows PowerShell 5.1 parser failure in the offline installer that
could stop installation before `.venv` was created. The failure was exposed when
users ran the documented command:

```powershell
powershell -ExecutionPolicy Bypass -File .\install_offline.ps1
```

## What Changed

### 1. Windows install guidance no longer trips the parser

The Windows offline and online installers no longer embed the next-step command
as `& ... cd <your-project> ...` inside a multiline PowerShell invocation. The
next-step command is now built from parser-safe text and avoids the
`<your-project>` placeholder that Windows PowerShell treats as syntax in some
contexts.

### 2. Windows release gates now use Windows PowerShell 5.1

Windows offline smoke, release build smoke, and published release artifact smoke
now invoke the installer through:

```powershell
WindowsPowerShell\v1.0\powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\install_offline.ps1
```

This matches the documented user path instead of only exercising PowerShell 7
(`pwsh`).

### 3. Regression tests lock the user-facing install path

Workflow tests now require Windows PowerShell 5.1 execution for Windows install
smoke. Installer tests also reject the parser-sensitive guidance pattern that
caused the `AmpersandNotAllowed` / `<` parser error.

### 4. Windows `.ps1` installers avoid legacy encoding traps

Windows PowerShell 5.1 can read UTF-8 scripts without a BOM using the system
ANSI code page, which can corrupt non-ASCII literals before parsing. The Windows
online and offline `.ps1` installers are now ASCII-only; localized guidance stays
in the Markdown user guide instead of inside the script source.

### 5. User guide separates offline and online install paths

`USER_GUIDE.zh-CN.md` now gives separate command blocks for packages that were
already downloaded next to the project and packages downloaded directly from
GitHub Release. The pre-downloaded path no longer includes `Invoke-WebRequest`,
so users do not get conflicting instructions.

## Compatibility Notes

- macOS / Linux use `.tar.gz` platform packages:
  - macOS Apple Silicon uses `ai-sdlc-offline-0.7.13-macos-arm64.tar.gz`
  - Linux x64 uses `ai-sdlc-offline-0.7.13-linux-amd64.tar.gz`
- Windows x64 uses `ai-sdlc-offline-0.7.13-windows-amd64.zip`
- Windows users on `0.7.12` should replace it with `0.7.13`; the failure happens
  before the package installs, so rerunning the `0.7.12` installer will not fix
  the issue.
- Users stuck on older releases, including CLIs that report
  `No such command 'install'`, should download the matching package from
  `releases/download/v0.7.13` and run `--upgrade-existing` once.
- After `0.7.13` is installed, normal future updates should return to
  `ai-sdlc self-update check`.
