# AI-SDLC v0.7.17 Release Notes

## Positioning

`0.7.17` is a Windows first-run onboarding hotfix on top of `0.7.16`.

It fixes the Windows path where users install with `-AddToPath` from a child PowerShell process, then immediately run bare `ai-sdlc init .` in the unchanged parent terminal and can accidentally invoke an older CLI that was already on PATH.

## What Changed

### 1. Windows installers make PATH consent and current-terminal behavior explicit

- `-AddToPath` remains the explicit consent signal for writing User PATH.
- Windows installers now print that no second confirmation prompt is shown because `-AddToPath` was provided.
- Windows installers now warn that the current parent terminal may still resolve an older `ai-sdlc`.
- Windows installers print `Get-Command ai-sdlc | Select-Object Source` so users can verify the actual command source.
- Windows installers print a direct `ai-sdlc.exe init .` shim as the immediate next command for the current terminal.
- Users who do not want PATH changes can still run the same commands through the bundled interpreter with `python -m ai_sdlc`.

### 2. Shell selection remains part of first init

`ai-sdlc init` continues to ask for both the AI agent entry and the project command shell in interactive terminals. The updated installer guidance reduces the chance that users run an older CLI that only asks for the agent entry.

### 3. Windows selector redraw no longer emits raw ANSI clear-screen codes

Interactive menu redraw now avoids writing `ESC[2J` / `ESC[H` to Windows terminals. This prevents PowerShell sessions that do not interpret those codes from showing repeated raw escape sequences while users navigate with arrow keys.

## Compatibility Notes

- Windows x64 uses `ai-sdlc-offline-0.7.17-windows-amd64.zip`
- macOS / Linux use `.tar.gz` bundles: `ai-sdlc-offline-0.7.17-macos-arm64.tar.gz` and `ai-sdlc-offline-0.7.17-linux-amd64.tar.gz`
- Users who want direct `ai-sdlc ...` commands should install with `-AddToPath` on Windows or `--add-to-path` on macOS/Linux.
- On Windows, after `powershell -File ... -AddToPath`, use the printed Direct shim in the current parent terminal or open a new terminal before using bare `ai-sdlc ...`.
- Users stuck on older releases, including CLIs that report `No such command 'install'`, should download the matching package from `releases/download/v0.7.17` and run `--upgrade-existing` once.
- After `0.7.17` is installed, normal future updates should return to `ai-sdlc self-update check`.
