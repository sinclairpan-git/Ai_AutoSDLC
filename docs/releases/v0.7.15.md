# AI-SDLC v0.7.15 Release Notes

## Positioning

`0.7.15` is a Windows offline install guidance hotfix on top of `0.7.14`.

It corrects the pre-downloaded offline package contract in the user guide. In
`0.7.14`, the pre-downloaded Windows command block still executed `cd ..` before
looking for the zip. If a user had already placed
`ai-sdlc-offline-0.7.14-windows-amd64.zip` in the current project directory, the
command moved to the parent directory and failed to find the package.

## What Changed

### 1. Pre-downloaded install commands use the current directory

The Windows, macOS, and Linux pre-downloaded guide blocks now state one rule:
place the archive in the current terminal directory, `cd` into that directory,
then run the command block. The command block checks only the current directory
and installs immediately when the expected archive is present.

The missing-package message is bilingual and explicit:

```text
Place the zip/tar.gz in the current directory, cd into that directory, then retry.
```

### 2. Online install commands remain separate

The online GitHub Release path is still documented as a separate scenario. It
may move to the project parent directory before downloading so the downloaded
bundle remains outside the business project. That rule no longer applies to the
pre-downloaded scenario.

### 3. Initialization commands are split by install scenario

The user guide now gives separate `init` commands after install:

- Scenario A: archive and extracted bundle are inside the current project
  directory.
- Scenario B: downloaded bundle is a sibling of the business project directory.

This prevents the install command from succeeding and the next `init` command
from using the wrong relative path.

## Compatibility Notes

- macOS / Linux use `.tar.gz` platform packages:
  - macOS Apple Silicon uses `ai-sdlc-offline-0.7.15-macos-arm64.tar.gz`
  - Linux x64 uses `ai-sdlc-offline-0.7.15-linux-amd64.tar.gz`
- Windows x64 uses `ai-sdlc-offline-0.7.15-windows-amd64.zip`
- Windows users on `0.7.12`, `0.7.13`, or `0.7.14` should replace it with
  `0.7.15` if they follow the pre-downloaded offline package instructions.
- Users stuck on older releases, including CLIs that report
  `No such command 'install'`, should download the matching package from
  `releases/download/v0.7.15` and run `--upgrade-existing` once.
- After `0.7.15` is installed, normal future updates should return to
  `ai-sdlc self-update check`.
