# AI-SDLC v0.9.1 Release Notes

## Positioning

`0.9.1` is a patch release for the Vue3 frontend governance line introduced in
`0.9.0`. It publishes the latest Vue3 enterprise frontend specification
alignment and closes the gap between the documented PrimeVue provider contract
and generated project output.

This release keeps `frontend_stack=vue3`, `provider_id=public-primevue`, and
`style_pack_id=modern-saas` as the ordinary first recommendation for new
frontend work, while preserving the advanced candidate matrix for experienced
users.

## What Changed

### 1. Latest Vue3 enterprise specification is now reflected in generated output

- The default `public-primevue` provider now carries the expected runtime
  contract for PrimeVue, `@primeuix/themes`, `primeicons`, Pinia, Vue Router,
  Axios, vee-validate, zod, and vue-i18n.
- The generated Vue3 entry loads `primeicons/primeicons.css` alongside the
  PrimeVue theme and application styles, so icon usage is not left to hidden
  downstream assumptions.
- Development dependencies now reflect the published quality baseline:
  `vue-tsc`, `@vitejs/plugin-vue`, `@antfu/eslint-config`, ESLint, Prettier,
  Vitest, Playwright, and related test tooling.

### 2. Frontend recommendation language is aligned with the provider policy

- `modern-saas` is described as a modern enterprise console style for PrimeVue
  token-first back-office, workspace, and admin scenarios.
- Enterprise/back-office keywords such as workbench, admin console, table,
  form, and approval flow remain Vue3 `public-primevue` default signals, not
  implicit Vue2 signals.
- Advanced mode still exposes style alternatives including `enterprise-default`,
  `data-console`, `high-clarity`, and `macos-glass`, plus the explicit
  `enterprise-vue2` compatibility path.

### 3. Regression guards now protect the published contract

- Unit and integration tests assert the PrimeVue dependency set, CSS import
  contract, beginner recommendation output, and advanced candidate wording.
- Release constraints now require the current release docs, user guide, offline
  bundle names, and workflow defaults to stay aligned with `v0.9.1`.
- The Windows install and upgrade gates remain part of the required validation
  set before publication.

## Compatibility Notes

- This is a patch release: it does not remove the Vue2 compatibility provider
  or change the confirmed advanced-selection path.
- Existing projects that explicitly select `enterprise-vue2` / Vue2 remain
  supported.
- `ai-sdlc init .` remains the ordinary setup entry. `adapter status` and
  `run --dry-run` remain troubleshooting commands, not beginner-path setup
  steps.
- `python -m ai_sdlc ...` remains available as the module-entry fallback when a
  shell has not refreshed PATH yet.

## Install And Upgrade

Use the platform release asset from
`https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/download/v0.9.1`.

- Windows: download `ai-sdlc-offline-0.9.1-windows-amd64.zip`, then run
  `install_offline.ps1 -AddToPath` for new installs or
  `install_offline.ps1 -UpgradeExisting` when replacing an older CLI.
- macOS / Linux: download the matching `.tar.gz` asset, unpack it, then run
  `install_offline.sh --add-to-path` for new installs or
  `--upgrade-existing` when replacing an older CLI.
- Existing users can try `ai-sdlc self-update check` first. If the output does
  not finish with version `0.9.1`, use the matching platform package and
  `--upgrade-existing`.
- If an older CLI reports `No such command 'install'`, skip diagnosis and use
  the matching platform package with `--upgrade-existing`.
