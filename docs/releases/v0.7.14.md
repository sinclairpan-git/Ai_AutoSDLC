# AI-SDLC v0.7.14 Release Notes

## Positioning

`0.7.14` is an offline install guidance patch on top of `0.7.13`.

It fixes a documentation-to-command mismatch in the user guide. The guide now
separates the pre-downloaded offline package path from the online GitHub Release
download path, and the pre-downloaded path gives users one strict placement rule:
put the package in the business project parent directory, `cd` into that parent
directory, then run the offline install commands.

## What Changed

### 1. Pre-downloaded offline install commands are explicit

The Windows, macOS, and Linux guide blocks now assume the archive is already in
the project parent directory. They check the current directory only and fail with
a bilingual message when the archive is missing:

```text
Place the zip/tar.gz in the project parent directory, cd into that directory, then retry.
```

This avoids hiding path mistakes behind Downloads/Desktop scanning or other
implicit path inference.

### 2. Online install commands stay separate

The online path still downloads directly from GitHub Release before extracting
and installing. It is documented as a different scenario, so users are not told
to pre-download a package and then given commands that ignore it.

### 3. Copy-paste blocks avoid terminating interactive POSIX shells

The macOS/Linux pre-downloaded snippets use a small shell function and `return 1`
for the missing-package branch. A path mistake stops the install attempt without
closing the user’s interactive terminal session.

## Compatibility Notes

- macOS / Linux use `.tar.gz` platform packages:
  - macOS Apple Silicon uses `ai-sdlc-offline-0.7.14-macos-arm64.tar.gz`
  - Linux x64 uses `ai-sdlc-offline-0.7.14-linux-amd64.tar.gz`
- Windows x64 uses `ai-sdlc-offline-0.7.14-windows-amd64.zip`
- Windows users on `0.7.12` or `0.7.13` should replace it with `0.7.14` if they
  follow the pre-downloaded offline package instructions.
- Users stuck on older releases, including CLIs that report
  `No such command 'install'`, should download the matching package from
  `releases/download/v0.7.14` and run `--upgrade-existing` once.
- After `0.7.14` is installed, normal future updates should return to
  `ai-sdlc self-update check`.
