# AI-SDLC v0.7.2 Release Notes

## Legacy Update Notice

If you are reading this page while trying to upgrade an older installed CLI, do
not follow this historical release's asset names or update behavior. Go to the
latest release and use the platform install package's upgrade mode:

- Latest release: `https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/tag/v0.7.8`
- Download and extract the matching package first:
  - macOS Apple Silicon: `ai-sdlc-offline-0.7.8-macos-arm64.tar.gz`
  - Linux x64: `ai-sdlc-offline-0.7.8-linux-amd64.tar.gz`
  - Windows x64: `ai-sdlc-offline-0.7.8-windows-amd64.zip`
- Then enter the extracted folder and run:
  - macOS / Linux: `./install_offline.sh --upgrade-existing`
  - Windows: `powershell -ExecutionPolicy Bypass -File .\install_offline.ps1 -UpgradeExisting`

After that one-time bridge, future updates should use `ai-sdlc self-update check`.

## Release Positioning

`0.7.2` is a runtime usability patch on top of `0.7.1`. It keeps the cloud-built offline asset and release-artifact smoke path from `0.7.1`, and adds a formal self-update / update-advisor surface for installed CLI users.

## What This Release Delivers

### 1. Installed runtimes can advise users about newer releases

- Installed `ai-sdlc` CLI runs can check the latest stable GitHub Release in a non-blocking way
- Source checkout, editable install, `uv run`, and `python -m ai_sdlc` development paths fail closed and stay quiet
- Results are cached with freshness and failure backoff so normal commands do not depend on the network

### 2. Explicit `self-update` helper commands

- `ai-sdlc self-update identity --json` reports installed-runtime identity and install-channel truth
- `ai-sdlc self-update evaluate --json` exposes machine-readable advisor state
- `ai-sdlc self-update check` provides a human-facing check command
- `ai-sdlc self-update instructions --version 0.7.2` prints platform-specific GitHub Release asset instructions
- The advisor does not silently download, overwrite, or replace the running CLI

### 3. Windows-safe advisor cache

- Update-advisor cache filenames now sanitize runtime identity strings for Windows compatibility
- The release includes a regression test for `sha256:` runtime identities, which previously exposed an invalid `:` filename on Windows

### 4. Release docs and workflow defaults are aligned

- `README.md`, `USER_GUIDE.zh-CN.md`, `packaging/offline/README.md`, `docs/pull-request-checklist.zh.md`, and `docs/框架自迭代开发与发布约定.md` now point at `v0.7.2`
- `release-build.yml` and `release-artifact-smoke.yml` default to `v0.7.2`
- Release consistency checks now require `docs/releases/v0.7.2.md` and the `0.7.2` offline asset names

## Boundaries

- `self-update` is an advisor and instruction surface; it is not a silent auto-updater
- `github-archive` installs can receive actionable GitHub Release asset instructions; other install channels receive lighter upstream notices until their channel truth is explicitly modeled
- Windows `.zip` and POSIX `.tar.gz` release assets remain platform-scoped and must be verified on their matching target platforms
