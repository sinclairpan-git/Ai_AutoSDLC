# AI-SDLC v0.9.5 Release Notes

## Positioning

`0.9.5` is the Vue3 visual-governance refresh release for the
`public-primevue` default provider line. It upgrades the default Vue3 enterprise
standard from v1.6.1 to the v1.7.1 AI-executable visual contract and carries the
same contract into generated scaffold artifacts, adapter guidance, and release
truth checks.

This is not a provider switch. The ordinary default remains
`frontend_stack=vue3`, `provider_id=public-primevue`, and
`style_pack_id=modern-saas`. The release makes the default PrimeVue route more
deterministic for business systems that need page-level structure, API contracts,
i18n, DTO transforms, and verifiable visual evidence.

## What Changed

### 1. Vue3 enterprise standard v1.7.1 is now canonical

- `docs/Vue3企业级前端开发规范方案.md` now carries the v1.7.1 AI-executable
  visual standard.
- The spec keeps Vue3, TypeScript, Vite, PrimeVue, `@primeuix/themes`,
  `primeicons`, UnoCSS, CSS Variables, Pinia, Vue Router, Axios,
  `vee-validate`, `zod`, and `vue-i18n` as the ordinary default stack.
- The visual contract now names PrimeVue semantic tokens, `theme.ts` as the
  single preset entry, Base / Business / Page layering, and path-level
  `pages` / API / transform conventions.

### 2. Managed Vue3 scaffold now exposes the landed structure

- New generated projects use `src/pages` for page components while keeping
  legacy `src/views` import-boundary scanning for compatibility.
- Managed delivery artifacts now include API client/interceptor/type skeletons,
  i18n initialization, and DTO transform entry points.
- The smoke route imports the generated page from `src/pages`, matching the
  refreshed visual contract instead of relying on view-only scaffolding.

### 3. Constraints and adapter instructions are aligned

- `verify constraints` checks the refreshed provider manifest, generated
  template paths, Vue3 frontend governance markers, and release truth surfaces.
- AGENTS / Codex / Claude Code / VS Code / Cursor adapter guidance keeps the
  same frontend confirmation defaults and advanced selection path.
- `program solution-confirm` still shows the beginner-friendly default first,
  while preserving explicit `vue2` / `enterprise-vue2` compatibility and
  advanced style-pack overrides.

## Compatibility Notes

- This release does not remove Vue2 compatibility or the advanced frontend
  solution-selection path.
- Vue3 `public-primevue` with `style_pack_id=modern-saas` remains the ordinary
  first recommendation for new frontend work.
- Existing projects using `src/views` continue to be scanned by the import
  boundary checks.
- `ai-sdlc init .` remains the beginner setup entry. `adapter status` and
  `run --dry-run` remain troubleshooting commands, not required beginner-path
  setup steps.
- `python -m ai_sdlc ...` remains available as the module-entry fallback when a
  shell has not refreshed PATH yet.

## Install And Upgrade

Use the platform release asset from
`https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/download/v0.9.5`.

- Windows: download `ai-sdlc-offline-0.9.5-windows-amd64.zip`, then run
  `install_offline.ps1 -AddToPath` for new installs or
  `install_offline.ps1 -UpgradeExisting` when replacing an older CLI.
- macOS / Linux: download the matching `.tar.gz` asset, unpack it, then run
  `install_offline.sh --add-to-path` for new installs or
  `--upgrade-existing` when replacing an older CLI.
- Existing users can try `ai-sdlc self-update check` first. If the output does
  not finish with version `0.9.5`, use the matching platform package and
  `--upgrade-existing`.
- If an older CLI reports `No such command 'install'`, skip diagnosis and use
  the matching platform package with `--upgrade-existing`.
