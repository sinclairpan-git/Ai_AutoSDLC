# AI-SDLC v0.8.9 Release Notes

## Positioning

`0.8.9` is a Windows upgrade-path patch after real cloud Windows E2E against
historical installs. It keeps the `0.8.8` user-facing update prompt behavior for
current and future installs, and fixes the installed module fallback path so
`python -m ai_sdlc self-update check` is no longer mistaken for a source
checkout when it comes from an installed distribution.

## What Changed

### 1. Installed module fallback is treated as installed runtime

- `python -m ai_sdlc ...` now stays eligible for installed-runtime update logic
  when `ai_sdlc/__main__.py` is inside the installed package distribution.
- Source checkouts, editable installs, and `uv run ai-sdlc ...` still stay quiet
  so framework development is not polluted by release notices.
- The regression is covered by `test_installed_module_invocation_is_installed_runtime`.

### 2. User guidance now matches real Windows E2E behavior

- `USER_GUIDE.zh-CN.md`, `README.md`, `packaging/offline/README.md`, release
  workflows, and release consistency guards now point to `v0.8.9`.
- The old-user upgrade chapter no longer assumes every historical `0.7.x` /
  `0.8.x` install can complete automatic self-update directly.
- If a legacy CLI only shows a release notice, does not ask the new `y/n`
  question, or does not finish with version `0.8.9`, the documented path is the
  platform package `--upgrade-existing` rescue upgrade.

### 3. Cloud Windows E2E evidence was added for historical update scenarios

- The Windows Update Prompt E2E workflow installs `v0.7.4` and `v0.8.0` in
  separate clean Windows jobs.
- It runs ordinary CLI commands through a Windows pseudo-terminal, checks
  whether the user-facing upgrade prompt appears, then records self-update
  results as artifacts.
- Current evidence showed those historical packages cannot be retroactively
  changed to provide the new prompt; users on those versions need one rescue
  package replacement before future updates can use the latest logic.

## Compatibility Notes

- No frontend provider, Vue3 PrimeVue, Vue2 enterprise, or managed delivery
  contract changes are introduced in this patch.
- `ai-sdlc --version` remains the supported version check in the current CLI and
  in release installers.
- `adapter status` and `run --dry-run` remain troubleshooting commands; normal
  setup should follow `ai-sdlc init .` and then return to the AI chat when the
  CLI says so.

## Install And Upgrade

Use the platform release asset from
`https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/download/v0.8.9`.

- Windows: download `ai-sdlc-offline-0.8.9-windows-amd64.zip`, then run
  `install_offline.ps1 -AddToPath` for new installs or
  `install_offline.ps1 -UpgradeExisting` when replacing an older CLI.
- macOS / Linux: download the matching `.tar.gz` asset, unpack it, then run
  `install_offline.sh --add-to-path` for new installs or
  `--upgrade-existing` when replacing an older CLI.
- Existing users can try `ai-sdlc self-update check` first. If the output does
  not finish with version `0.8.9`, use the matching platform package and
  `--upgrade-existing`.
- If an older CLI reports `No such command 'install'`, skip diagnosis and use
  the matching platform package with `--upgrade-existing`.
