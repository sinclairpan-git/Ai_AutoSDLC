# AI-SDLC v0.7.19 Release Notes

## Positioning

`0.7.19` is an AgentOps governance patch focused on separating personal AI-SDLC use from enterprise-required AgentOps monitoring.

Personal and single-machine users remain on the local-first path: no AgentOps setup is required, no token is needed, and normal `ai-sdlc init .` / `ai-sdlc run` usage stays quiet unless the user or project explicitly opts into reporting.

Enterprise and project-required configurations now fail closed more consistently when AgentOps is required but misconfigured, downgraded, redirected, or returns diagnostic receipts.

## What Changed

### 1. Personal use stays local by default

- The default reporting mode remains `off`.
- A plain personal checkout does not need AgentOps endpoint, token, profile, or enterprise setup.
- Enterprise setup guidance is kept out of the ordinary quick-start path and linked separately.

### 2. Managed enterprise profiles are authoritative

- Required enterprise profiles cannot be downgraded by lower-trust process environment values.
- Explicit enterprise profile paths now fail closed when missing or malformed.
- Profile-defined endpoint, ingestion mode, and token environment selection take precedence over process-local overrides.
- `ai-sdlc enterprise configure --reporting-mode off` can write an off profile without requiring a dummy endpoint.

### 3. Project-required AgentOps cannot be bypassed by env downgrades

- Project-level `agentops_reporting_mode: required` now wins over environment values such as `AGENTOPS_REPORTING_MODE=off`.
- Project-required endpoint, ingestion mode, and token environment selection are honored before process environment overrides.
- Required runs still block on missing endpoint/token diagnostics when the project declares AgentOps reporting mandatory.

### 4. Required delivery blocks on diagnostic receipts

- Non-success receipt states, including rejected outbox states with zero or omitted rejection counters, are treated as diagnostics.
- Required mode exits nonzero when AgentOps delivery returns diagnostic receipt states.
- Diagnostic CLI output now includes the persisted receipt summary path and the first stale/rejected/DLQ item code, message, and retry guidance when available.
- Diagnostic item selection prefers the actual stale/rejected/DLQ item before falling back to non-diagnostic metadata.

## Compatibility Notes

- Windows x64 uses `ai-sdlc-offline-0.7.19-windows-amd64.zip`
- macOS / Linux use `.tar.gz` bundles: `ai-sdlc-offline-0.7.19-macos-arm64.tar.gz` and `ai-sdlc-offline-0.7.19-linux-amd64.tar.gz`
- Users who want direct `ai-sdlc ...` commands should install with `-AddToPath` on Windows or `--add-to-path` on macOS/Linux.
- Users who cannot rely on PATH can keep using `python -m ai_sdlc ...` from the installed runtime.
- Personal users should keep using the release installer, company package, or `ai-sdlc self-update check`. Do not manually create Python venvs or install pip dependencies unless the installer or `ai-sdlc doctor` asks for it.
- Enterprise AgentOps setup remains documented separately in `docs/enterprise-agentops-setup.zh-CN.md`.
- Existing `v0.7.18` users can upgrade directly to `v0.7.19` with the matching platform package or `ai-sdlc self-update check`.
- Users stuck on older releases, including CLIs that report `No such command 'install'`, should download the matching package from `releases/download/v0.7.19` and run `--upgrade-existing` once.
