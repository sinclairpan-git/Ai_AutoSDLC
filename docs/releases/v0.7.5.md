# AI-SDLC v0.7.5 Release Notes

## Release Positioning

`0.7.5` is a beginner-onboarding and documentation guardrail patch on top of `0.7.4`. It publishes the init-first UX correction that keeps ordinary users on one clear path: install or update AI-SDLC, run `ai-sdlc init .`, then switch to the AI chat when the CLI says setup is complete.

## What This Release Delivers

### 1. Init-first beginner path is now the published contract

- `README.md` and `USER_GUIDE.zh-CN.md` now describe the real 0-to-1 path around `ai-sdlc init .`
- `adapter status` and `run --dry-run` remain available as troubleshooting commands, but are no longer presented as beginner setup steps
- Beginner guidance emphasizes bilingual `当前结果 / Result` and `下一步 / Next` output instead of internal governance state translation

### 2. Generated AI-agent instructions match the same path

- Codex, Claude Code, Cursor, and VS Code adapter templates now tell agents to start ordinary users with `init`
- Generated instructions explicitly avoid asking users to manually install Python, venv, pip, or dependencies unless the CLI reports an actionable error
- When `init` says to switch to AI chat, generated instructions tell the agent to continue requirement clarification and implementation rather than asking for extra terminal preflight commands

### 3. Documentation drift is now blocked by constraints

- `ai-sdlc verify constraints` now checks README, `USER_GUIDE.zh-CN.md`, root `AGENTS.md`, and adapter template sources for the current init-first markers
- The verifier blocks old manual setup wording such as mandatory `adapter status` plus `run --dry-run` onboarding chains
- Framework source checkouts now fail constraints if adapter template sources are present but root `AGENTS.md` is missing

### 4. Release docs and workflow defaults are aligned

- `README.md`, `USER_GUIDE.zh-CN.md`, `packaging/offline/README.md`, `docs/pull-request-checklist.zh.md`, and `docs/框架自迭代开发与发布约定.md` now point at `v0.7.5`
- `release-build.yml` and `release-artifact-smoke.yml` default to `v0.7.5`
- Release consistency checks now require `docs/releases/v0.7.5.md` and the `0.7.5` offline asset names

## Boundaries

- `0.7.5` does not change the offline asset platform split: Windows uses `.zip`; macOS and Linux use `.tar.gz`
- `0.7.5` does not remove advanced diagnostic commands; it changes when they are shown to ordinary users
- Existing user-customized adapter files are still preserved by adapter installation

## Release Verification

Required before marking `v0.7.5` complete:

- `uv run pytest -q`
- `uv run ruff check src tests`
- `uv run ai-sdlc verify constraints`
- `python -m ai_sdlc program truth sync --dry-run`
- To build and upload release assets from cloud runners: `release-build.yml` for tag `v0.7.5`
- After publishing the GitHub Release assets: `release-artifact-smoke.yml` for tag `v0.7.5`
