# AI-SDLC v0.7.18 Release Notes

## Positioning

`0.7.18` is a production-feedback patch focused on making AI-SDLC harder to bypass and easier to adopt in existing projects.

It addresses four recurring field issues:

- AI agents generating code directly from `spec.md` without an executable task.
- IDE adapter readiness being treated as a hard host verification requirement even when the host cannot provide machine-verifiable load evidence.
- Poor comment and text-quality behavior, including deleted original comments and Chinese mojibake.
- Existing projects with their own JSON/Markdown task progress being difficult to connect to AI-SDLC mid-stream.

## What Changed

### 1. Code changes now require an executable task

- Added parser support for executable task metadata in `tasks.md`.
- Added task preparation and guard checks that block code changes when the current work item has no valid executable task.
- Guard output now includes actionable candidate fixes instead of relying on unverifiable IDE state.

### 2. Adapter verification is diagnostic, not the main gate

- Main user paths no longer block just because an IDE cannot prove `verified_loaded`.
- `adapter status` and beginner guidance now direct users back to AI chat and the current executable task instead of asking for routine dry-runs.
- Diagnostic states remain available for troubleshooting.

### 3. Comment and text quality policies

- Added comment-language policy: new comments follow the current or recent user communication language, defaulting to Simplified Chinese.
- Added original-comment deletion protection in `verify constraints`.
- Added diff-based UTF-8 / mojibake checks for changed text, with allowlist and Traditional Chinese project support.
- Added rules requiring useful comments for complex intent, boundaries, compatibility, concurrency, cache, and error handling without restating obvious code.

### 4. Brownfield adoption flow

- Added `ai-sdlc adopt .` for 接入已有项目.
- The command scans existing JSON/Markdown task files, README/TODO sources, git branch/diff/commit evidence, tests, and project structure within bounded budgets.
- It writes bridge artifacts under `.ai-sdlc/adoption/` without modifying original task files.
- Users can correct the recommended continuation point with natural language, for example `ai-sdlc adopt . --prefer "支付回调"`.

## Compatibility Notes

- Windows x64 uses `ai-sdlc-offline-0.7.18-windows-amd64.zip`
- macOS / Linux use `.tar.gz` bundles: `ai-sdlc-offline-0.7.18-macos-arm64.tar.gz` and `ai-sdlc-offline-0.7.18-linux-amd64.tar.gz`
- Users who want direct `ai-sdlc ...` commands should install with `-AddToPath` on Windows or `--add-to-path` on macOS/Linux.
- Users who cannot rely on PATH can keep using `python -m ai_sdlc ...` from the installed runtime.
- Existing projects can continue using `ai-sdlc init .`; use `ai-sdlc adopt .` after init only when the project already has external task progress to bridge.
- Users stuck on older releases, including CLIs that report `No such command 'install'`, should download the matching package from `releases/download/v0.7.18` and run `--upgrade-existing` once.
- After `0.7.18` is installed, normal future updates should return to `ai-sdlc self-update check`.

## Verification

Required before marking `v0.7.18` complete:

- `uv run pytest`
- `uv run ruff check src tests`
- `uv run ai-sdlc verify constraints`
- GitHub PR checks for this release branch
- Codex review with no unresolved blocking findings
- `release-build.yml` for tag `v0.7.18`
- `release-artifact-smoke.yml` for tag `v0.7.18`
