# AI-SDLC v0.7.10 Release Notes

## Positioning

`0.7.10` is an update-advisor visibility patch on top of `0.7.9`.

It closes the remaining production gap in the installed-runtime update path:
the framework had code for "a newer version is available" notices, but the
notice could feel absent in real use because automatic rendering immediately
recorded the notice as acknowledged. If the user missed the first display, the
same version would not be shown again.

The intended behavior is now:

- automatic update notices are throttled, not treated as user acknowledgement
- a known newer version remains actionable while the cache is `stale_but_usable`
- a temporary refresh failure does not hide a previously known newer version
- explicit acknowledgement remains available for hosts that intentionally want
  to suppress a notice version

## What Changed

### 1. Rendering is no longer acknowledgement

Automatic CLI notices now record `last_rendered_at` and
`rendered_notice_version` only. They no longer write `last_acknowledged_at`.
This keeps repeated commands from spamming the user while avoiding the old
"shown once, gone forever" behavior.

### 2. Notices repeat after a short throttle window

The CLI suppresses repeated automatic notices for the same version for a short
window, then allows the reminder to appear again. The current throttle is six
hours.

### 3. Stale-but-usable cache can still remind users

If the installed runtime already has cached release truth showing a newer
version, AI-SDLC can continue to show the actionable update notice while the
cache is still usable, even when the next lightweight refresh is slow or
temporarily unavailable.

## Compatibility Notes

- macOS / Linux use `.tar.gz` platform packages:
  - macOS Apple Silicon uses `ai-sdlc-offline-0.7.10-macos-arm64.tar.gz`
  - Linux x64 uses `ai-sdlc-offline-0.7.10-linux-amd64.tar.gz`
- Windows x64 uses `ai-sdlc-offline-0.7.10-windows-amd64.zip`
- Users already stuck on `0.7.7`, `0.7.8`, or `0.7.9` with update-check
  trouble, including `No such command 'install'`, should download the matching
  package from `releases/download/v0.7.10` and run `--upgrade-existing` once.
- After `0.7.10` is installed, normal future updates should return to
  `ai-sdlc self-update check`.
