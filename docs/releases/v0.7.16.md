# AI-SDLC v0.7.16 Release Notes

## Positioning

`0.7.16` is an install entrypoint and release-consistency hardening patch on top of `0.7.15`.

It fixes the mismatch between documented bare `ai-sdlc ...` commands and installers that did not explicitly place the CLI entrypoint on the user PATH. It also closes the process gap that allowed local package versions, docs, release notes, and workflow defaults to drift behind published GitHub releases.

## What Changed

### 1. PATH opt-in is explicit and tested

- Windows installers support `-AddToPath`.
- macOS and Linux installers support `--add-to-path`.
- Installers update the current shell/session path where possible and print the exact next command.
- Published-release smoke tests now install with PATH opt-in and verify the bare command resolves with `Get-Command ai-sdlc` or `command -v ai-sdlc`.

### 2. No-PATH fallback remains a first-class path

The user guide and offline package guide now spell out the rule:

- If PATH was written, users run `ai-sdlc --help`, `ai-sdlc init .`, `ai-sdlc adapter status`, and `ai-sdlc run --dry-run`.
- If PATH was not written, users replace the `ai-sdlc` prefix with the package Python module entry, such as the bundled venv interpreter plus `python -m ai_sdlc`.

### 3. Release version truth is gated

`verify constraints` now checks that package version, current release note, release entry docs, workflow defaults, and platform asset names all use the staged release version.

## Compatibility Notes

- Windows x64 uses `ai-sdlc-offline-0.7.16-windows-amd64.zip`
- macOS / Linux use `.tar.gz` bundles: `ai-sdlc-offline-0.7.16-macos-arm64.tar.gz` and `ai-sdlc-offline-0.7.16-linux-amd64.tar.gz`
- Users who want direct `ai-sdlc ...` commands should install with `-AddToPath` on Windows or `--add-to-path` on macOS/Linux.
- Users who do not want PATH changes can still run the same commands through the bundled interpreter with `python -m ai_sdlc`.
- Users stuck on older releases, including CLIs that report `No such command 'install'`, should download the matching package from `releases/download/v0.7.16` and run `--upgrade-existing` once.
- After `0.7.16` is installed, normal future updates should return to `ai-sdlc self-update check`.

## Verification

Required before marking `v0.7.16` complete:

- `bash -n packaging/offline/install_offline.sh packaging/install_online.sh`
- PowerShell parser check for `packaging/offline/install_offline.ps1` and `packaging/install_online.ps1`
- `uv run pytest tests/integration/test_offline_bundle_scripts.py tests/integration/test_github_workflows.py tests/unit/test_verify_constraints.py tests/unit/test_packaging_backend.py -q`
- `uv run ai-sdlc verify constraints`
- `uv run pytest -q`
- `uv run ruff check src tests packaging`
- `uv build`
- `release-build.yml` for tag `v0.7.16`
- `release-artifact-smoke.yml` for tag `v0.7.16`
