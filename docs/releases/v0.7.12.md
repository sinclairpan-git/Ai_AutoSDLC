# AI-SDLC v0.7.12 Release Notes

## Positioning

`0.7.12` is an offline runtime validation patch on top of `0.7.11`.

It publishes the follow-up fixes from the offline runtime release gate: release
assets must be built from uv-managed Python runtimes, and Windows test fixtures
now model the hosted Python DLL layout used by GitHub Actions.

## What Changed

### 1. Release workflows require uv-managed Python

Release build, POSIX offline smoke, and Windows offline smoke workflows now use
`uv python find --managed-python 3.11` after installing the runtime. This keeps
the packaged `python-runtime/` rooted in uv's managed interpreter instead of
accidentally resolving to a system or toolcache Python.

### 2. Windows runtime fixtures find the real DLL root

Windows compatibility tests now search the venv executable, base executable,
`sys.base_prefix`, and `sys.exec_prefix` for `python3xx.dll`. This matches the
layout used by hosted Windows Python, where the DLL can live beside the base
runtime rather than beside `.venv\Scripts\python.exe`.

### 3. Cross-platform gates covered the fixed path

The release gate matrix was rerun after the workflow fix and passed across
Windows, macOS, and Linux for Python 3.11 / 3.12 compatibility, plus the offline
smoke workflows that install and verify bundled runtimes.

## Compatibility Notes

- macOS / Linux use `.tar.gz` platform packages:
  - macOS Apple Silicon uses `ai-sdlc-offline-0.7.12-macos-arm64.tar.gz`
  - Linux x64 uses `ai-sdlc-offline-0.7.12-linux-amd64.tar.gz`
- Windows x64 uses `ai-sdlc-offline-0.7.12-windows-amd64.zip`
- Users stuck on older releases, including CLIs that report
  `No such command 'install'`, should download the matching package from
  `releases/download/v0.7.12` and run `--upgrade-existing` once.
- After `0.7.12` is installed, normal future updates should return to
  `ai-sdlc self-update check`.

## Verification

Required before marking `v0.7.12` complete:

- `uv run pytest tests/integration/test_offline_bundle_scripts.py tests/integration/test_github_workflows.py -q`
- `uv run pytest -q`
- `uv run ruff check src tests packaging/offline/verify_offline_bundle.py`
- `uv run ai-sdlc verify constraints`
- `uv build`
- `release-build.yml` for tag `v0.7.12`
- `release-artifact-smoke.yml` for tag `v0.7.12`
