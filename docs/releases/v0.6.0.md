# AI-SDLC v0.6.0 Release Notes

## 版本定位

`0.6.0` 是一次面向前端框架治理能力的阶段性版本发布。它不是单纯的 bugfix，也不是完整 P2 质量平台，而是把这批 P1 前端治理成果正式收束成一个可以对外使用、对内继续演进的新版本。

这次发布的重点是：

- 把前端治理从“经验式约束”推进到“Contract / Kernel / Gate / Verify / Recheck”统一真值链路
- 冻结前端 diagnostics contract，明确 `missing_artifact / invalid_artifact / valid_empty / drift / clean` 的互斥语义
- 冻结 P1 visual / a11y foundation 最小质量面，建立状态可见性、基础语义、键盘可达性与焦点连续性的最低门槛
- 让 CLI、`ProgramService`、gate surface 与文档口径一致，不再各自重算前端质量语义

## 本版本实现了什么

### 1. 前端诊断语义收口

`066` 把 frontend verification 的诊断层拆成了独立 truth：

- `Layer A`: Verification Truth
- `Layer B`: Diagnostics Truth
- `Layer C`: Surface Adapters

这次发布后，前端结果不再只是“有问题 / 没问题”的粗粒度状态，而是能稳定区分：

- `missing_artifact`
- `invalid_artifact`
- `valid_empty`
- `drift`
- `clean`

这意味着 verify、gate、CLI、`ProgramService` 读到的是同一份真值，而不是各自从空列表、异常字符串或路径缺失里猜语义。

### 2. UI Kernel 与 page recipe 语义正式成型

P1 已把前端统一协议层推进到可用状态：

- `Ui*` 语义组件协议被扩展并收口
- page recipe 标准本体被冻结
- `DashboardPage / DialogFormPage / SearchListPage / WizardPage` 等标准页型进入统一约束面

这让前端开发可以围绕“页面应该有哪些语义块、状态块和交互底线”来实现，而不是每个页面都重新定义结构。

### 3. P1 visual / a11y foundation 冻结

`071` 把 P1 的基础视觉与可访问性门槛冻结下来，覆盖：

- 状态可见性
- 关键区域可感知性
- 受控容器连续性
- 错误与状态可感知性
- 基础语义 / 命名
- 键盘可达性
- 焦点连续性

这不是完整视觉回归平台，也不是完整 WCAG 平台，而是前端体验质量的最低可交付线。

### 4. gate / verify / CLI / service 口径对齐

这次发布把前端质量判断统一到一条链路上：

- CLI 不再自己猜前端诊断语义
- `ProgramService` 不再重新解释 gap / drift / empty
- gate surface 只消费已冻结 truth
- 文档、测试与实现的表达保持同一套口径

## 这次发布带来的价值

- 前端开发更容易按协议落地，不容易跑偏到私有 DOM、临时截图脚本或局部规则
- 质量判断更诚实，能明确区分“没证据”“坏证据”“合法空输入”“真实漂移”
- 反馈闭环更短，问题能更直接回到统一 truth，而不是在多个 surface 之间来回对账
- 对存量 Vue2 企业项目更友好，可以白名单、适配器、受控 recipe 的方式渐进接入
- 后续 P2 的 modern provider、多主题、多风格、完整质量平台，都能在这个版本的真值基线之上继续演进

## 需要明确的边界

`0.6.0` 已经是一个可以发布的阶段性版本，但它仍然不是终局。

当前版本已经具备：

- 前端 Contract / Kernel / Diagnostics / Gate 的统一真值链路
- P1 visual / a11y foundation 的最低质量门槛
- 可复用的 surface adapter 语义

当前版本仍未提供：

- 完整 visual regression 平台
- 完整 a11y 审计平台
- modern provider
- multi-theme / multi-style 终局能力
- 全量前端 runtime 重构

换句话说，`0.6.0` 发布的是“可治理的前端框架基线”，不是“终局版前端平台”。
