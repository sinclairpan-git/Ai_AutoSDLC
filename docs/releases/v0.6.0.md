# AI-SDLC v0.6.0 Release Notes

## Release Positioning

`0.6.0` is a staged release focused on frontend governance capabilities. It is not a simple bugfix release, and it is not the full P2 quality platform. Instead, it formally consolidates the P1 frontend governance work into a version that is ready for external use and internal evolution.

The main goals of this release are:

- Move frontend governance from heuristic rules to a unified `Contract / Kernel / Gate / Verify / Recheck` truth chain
- Freeze the frontend diagnostics contract and define the mutually exclusive meanings of `missing_artifact`, `invalid_artifact`, `valid_empty`, `drift`, and `clean`
- Freeze the P1 visual and accessibility foundation as the minimum quality baseline for state visibility, semantic structure, keyboard reachability, and focus continuity
- Align CLI, `ProgramService`, gate surfaces, and documentation so they consume the same frontend quality truth

## What This Release Delivers

### 1. Frontend diagnostic semantics are now formalized

`066` split the frontend verification diagnostics into independent truth layers:

- `Layer A`: Verification Truth
- `Layer B`: Diagnostics Truth
- `Layer C`: Surface Adapters

With this release, frontend outcomes are no longer treated as a coarse "problem / no problem" state. They can now be reliably distinguished as:

- `missing_artifact`
- `invalid_artifact`
- `valid_empty`
- `drift`
- `clean`

This means verify, gate, CLI, and `ProgramService` all read the same truth instead of inferring semantics from empty lists, exception strings, or missing paths.

### 2. UI Kernel and page recipe semantics are now stable

P1 brought the shared frontend protocol layer into a usable state:

- `Ui*` semantic component protocols were expanded and consolidated
- The page recipe standard was frozen
- Standard page shapes such as `DashboardPage`, `DialogFormPage`, `SearchListPage`, and `WizardPage` now live under a unified contract

This lets frontend developers build around the semantic blocks, state blocks, and interaction baseline a page should have instead of redefining structure for every view.

### 3. The P1 visual and accessibility foundation is frozen

`071` locked the minimum visual and accessibility baseline for P1, covering:

- State visibility
- Visibility of key regions
- Controlled container continuity
- Error and status perceptibility
- Basic semantics and naming
- Keyboard reachability
- Focus continuity

This is not a complete visual regression platform and not a full WCAG auditing system. It is the minimum delivery line for frontend experience quality.

### 4. gate / verify / CLI / service semantics are aligned

This release puts frontend quality judgment onto one shared path:

- CLI no longer guesses frontend diagnostic semantics independently
- `ProgramService` no longer reinterprets gap / drift / empty states
- Gate surfaces only consume frozen truth
- Documentation, tests, and implementation now speak the same language

## Value Delivered by This Release

- Frontend developers can implement against a shared protocol instead of drifting into private DOM shapes, ad hoc screenshot scripts, or local-only rules
- Quality decisions are more honest because the system can separate "no evidence", "bad evidence", "legitimate empty input", and "real drift"
- Feedback loops are shorter because issues flow back to the unified truth layer instead of being reconciled across multiple surfaces
- Existing Vue 2 enterprise projects can adopt the framework incrementally through whitelisted providers, adapters, and controlled recipes
- Future P2 work such as modern providers, multi-theme / multi-style support, and a full quality platform can evolve on top of this truth baseline

## Boundaries

`0.6.0` is a releasable staged version, but it is not the endpoint.

What this release already provides:

- A unified truth chain across Contract / Kernel / Diagnostics / Gate
- The P1 visual and accessibility minimum baseline
- Reusable surface adapter semantics

What this release does not yet provide:

- A complete visual regression platform
- A complete accessibility audit platform
- A modern provider
- A full multi-theme / multi-style end state
- A full frontend runtime rewrite

In short, `0.6.0` ships a governable frontend framework baseline, not a final platform.
