# AI-SDLC v0.7.6 Release Notes

## Positioning

`0.7.6` is a self-update continuity patch on top of `0.7.5`. It fixes the real-world failure where an installed CLI detected a newer GitHub Release but still left ordinary users with either vague channel guidance or a command that older installs did not support.

The intended user path is now:

```bash
ai-sdlc self-update check
```

For current and future installed runtimes, that one command checks for a newer release, downloads the matching release asset, installs it into the current runtime, and verifies the installed version. Users should not need to copy a second update command during the normal path.

## What Changed

### 1. `self-update check` now performs the update

- `ai-sdlc self-update check` no longer stops after rendering advisory text when an actionable update exists
- The command now calls the existing self-update installer automatically
- JSON mode remains evaluation-only so scripts can still inspect update state without mutating the runtime
- Successful updates still end with a bilingual result panel and no follow-up command

### 2. Unknown installed channels can still use the release updater

- Installed non-editable runtimes are now eligible for the GitHub Release asset updater even when their original install channel cannot be classified
- The old fallback text, "current install channel is not confirmed for automatic update", is no longer the normal installed-runtime path
- Source checkouts, editable installs, and module-style development runs still fail closed and stay quiet

### 3. Legacy updater gap is documented as a one-time rescue path

Some pre-`0.7.6` installs may report `No such command 'install'` because they do not have the `self-update install` subcommand at all. Documentation now calls that out as a legacy break and gives one rescue upgrade command per shell family. After crossing that gap, users return to `ai-sdlc self-update check`.

macOS / Linux, activated venv:

```bash
"$(dirname "$(command -v ai-sdlc)")/python" -m pip install --upgrade --force-reinstall "https://github.com/sinclairpan-git/Ai_AutoSDLC/archive/refs/tags/v0.7.6.tar.gz"
```

Windows PowerShell, activated venv:

```powershell
& (Join-Path (Split-Path (Get-Command ai-sdlc).Source) "python.exe") -m pip install --upgrade --force-reinstall "https://github.com/sinclairpan-git/Ai_AutoSDLC/archive/refs/tags/v0.7.6.zip"
```

## Compatibility Notes

- Offline release assets remain platform-scoped: Windows uses `.zip`; macOS and Linux use `.tar.gz`
- `self-update evaluate --json` and `self-update check --json` do not mutate the install
- The first automatic update from very old versions may still require the documented one-time rescue command because those old binaries cannot gain a missing subcommand retroactively
