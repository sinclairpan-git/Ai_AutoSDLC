# AI-SDLC v0.8.7 Release Notes

## Positioning

`0.8.7` is a user-upgrade patch for `0.7.6+` and `0.8.x` installs. It focuses on
making update, version verification, and already-initialized project guidance
behave like product flows instead of framework diagnostics.

## What Changed

### 1. `self-update check` can upgrade unknown-channel installs

- Existing `0.7.6+` installs that are detected as an unknown channel can now
  continue into the automatic download, install, and version-verification flow.
- Windows Git Bash / MINGW64 users no longer get stuck at "new release found"
  when the current installation can be updated safely.
- The updater keeps beginner-facing output focused on success or a single
  retry path; internal channel and PATH diagnostics stay out of ordinary output.

### 2. `ai-sdlc --version` is now a supported global verification command

- `ai-sdlc --version` and `python -m ai_sdlc --version` both report the current
  installed package version.
- Installers and user guidance can use the same command as the post-install
  success check.

### 3. Windows PATH repair prefers the latest CLI without exposing conflicts

- The updater and Windows installers now prefer the newly installed AI-SDLC
  command directory so `ai-sdlc` resolves to the latest version.
- Shared Python or pipx script directories are preserved; AI-SDLC no longer
  removes an entire PATH entry just because it contains an older `ai-sdlc` shim.
- If the user invokes a supported module entrypoint without a bare `ai-sdlc`
  command on PATH, the update can still finish after verifying the installed
  package version.

### 4. `init .` no longer routes ordinary users into adapter diagnostics

- For an already-initialized project, `init .` no longer prints `adapter
  ingress truth`, `host-ingress proof`, or `run --dry-run` as the normal next
  step.
- The ordinary next step is to return to the AI chat or select the AI entry only
  when setup actually needs that choice.
- Raw adapter diagnostics remain available from explicit troubleshooting
  commands and machine-readable outputs.

## Compatibility Notes

- No frontend provider, scaffold, workflow, or Vue3 PrimeVue default-provider
  contract changes are introduced in this patch.
- Existing projects that selected `enterprise-vue2` or `public-primevue` keep
  their selected frontend provider behavior.
- Machine-readable diagnostic fields keep their existing internal names for
  framework and CI consumers.

## Install And Upgrade

Use the platform release asset from
`https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/download/v0.8.7`.

- Windows: download `ai-sdlc-offline-0.8.7-windows-amd64.zip`, then run
  `install_offline.ps1 -AddToPath` for new installs or
  `install_offline.ps1 -UpgradeExisting` when replacing an older CLI.
- macOS / Linux: download the matching `.tar.gz` asset, unpack it, then run
  `install_offline.sh --add-to-path` for new installs or
  `--upgrade-existing` when replacing an older CLI.
- Existing `0.7.6+` users can normally run `ai-sdlc self-update check` to get
  this release directly.
- If an older installed CLI reports `No such command 'install'`, replace it
  once with the matching platform asset and `--upgrade-existing`; after that,
  use `ai-sdlc self-update check` for future releases.

Release assets are produced by `release-build.yml`. After the GitHub Release is
published and assets are attached, `release-artifact-smoke.yml` verifies the
published `.zip` and `.tar.gz` assets from the release page.
