# AI-SDLC v0.7.4 Release Notes

## Legacy Update Notice

If you are reading this page while trying to upgrade an older installed CLI, do
not follow this historical release's asset names or update behavior. Go to the
latest release and use the platform install package's upgrade mode:

- Latest release: `https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/tag/v0.7.14`
- Download and extract the matching package first:
  - macOS Apple Silicon: `ai-sdlc-offline-0.7.14-macos-arm64.tar.gz`
  - Linux x64: `ai-sdlc-offline-0.7.14-linux-amd64.tar.gz`
  - Windows x64: `ai-sdlc-offline-0.7.14-windows-amd64.zip`
- Then enter the extracted folder and run:
  - macOS / Linux: `./install_offline.sh --upgrade-existing`
  - Windows: `powershell -ExecutionPolicy Bypass -File .\install_offline.ps1 -UpgradeExisting`

After that one-time bridge, future updates should use `ai-sdlc self-update check`.

## Release Positioning

`0.7.4` is a self-update UX patch on top of `0.7.3`. It closes the gap where update guidance still asked users to manually download release assets, extract archives, and run installer scripts. The intended user path is now direct: run one update command and get a clear bilingual success or failure result.

## What This Release Delivers

### 1. Automatic self-update install

- `ai-sdlc self-update install --version <version>` now downloads the matching GitHub Release asset, extracts it, installs the target wheel into the current runtime, and verifies the installed version
- Successful updates end with `更新完成 / Update completed` and do not ask the user to run follow-up commands
- Failed updates show the error and a single retry command
- Windows launcher upgrades re-execute through `python -m ai_sdlc` before pip replaces console scripts, avoiding locked `ai-sdlc.exe` failures

### 2. Safer release asset handling

- Tar extraction rejects symlink, hardlink, unsafe path, and unsupported member entries before unpacking
- Zip extraction validates target paths before unpacking
- The installer selects the exact `ai_sdlc-<target-version>-*.whl` wheel and fails clearly if it is missing or ambiguous

### 3. User-facing update guidance is no longer a manual recipe

- `self-update instructions` now points users to the automatic install command instead of printing `curl`, `tar`, or `install_offline` steps
- Update-advisor output now recommends `ai-sdlc self-update install --version <version>`
- Online/offline installer output and beginner guidance keep the bilingual `当前结果 / Result` and `下一步 / Next` format

### 4. Release docs and workflow defaults are aligned

- `README.md`, `USER_GUIDE.zh-CN.md`, `packaging/offline/README.md`, `docs/pull-request-checklist.zh.md`, and `docs/框架自迭代开发与发布约定.md` now point at `v0.7.4`
- `release-build.yml` and `release-artifact-smoke.yml` default to `v0.7.4`
- Release consistency checks now require `docs/releases/v0.7.4.md` and the `0.7.4` offline asset names

## Boundaries

- The automatic updater still requires network access to GitHub Releases
- If the current environment cannot replace its installed package, the command fails loudly instead of pretending the update completed
- Offline release assets remain platform-scoped: Windows `.zip`, macOS / Linux `.tar.gz`
