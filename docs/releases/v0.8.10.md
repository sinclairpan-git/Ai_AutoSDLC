# AI-SDLC v0.8.10 Release Notes

## Positioning

`0.8.10` is a Windows command-entry patch after user feedback from an online
release installation in Git Bash / MINGW64. The install itself succeeded, but
bare `ai-sdlc --version` still resolved a stale `Python\Scripts` entry. This
patch makes the Windows installer own a stable user-level command entry instead
of depending on whichever Python `Scripts` directory appears first on PATH.

## What Changed

### 1. Windows installers create a stable AI-SDLC command shim

- `install_offline.ps1 -AddToPath` and `install_online.ps1 -AddToPath` now
  create `%LOCALAPPDATA%\AI-SDLC\bin`.
- The shim directory contains `ai-sdlc.exe` and `ai-sdlc.cmd`; installers remove
  any legacy same-basename `ai-sdlc.ps1` from that PATH directory so PowerShell
  does not resolve an unsigned script before the executable.
- The shim directory is written to User PATH before older entries, so new
  PowerShell, cmd, and Git Bash terminals resolve the latest installed CLI.

### 2. Git Bash startup is repaired for new sessions

- Windows installers update `.bashrc` and the active Git Bash login profile
  (`.bash_profile`, `.bash_login`, or `.profile`) with a managed PATH block for
  the stable shim directory.
- The managed block runs `hash -r` so new Git Bash sessions do not keep a stale
  command resolution.
- Existing already-open parent shells still cannot be mutated by a child
  PowerShell process; the installer continues to print a full direct command for
  immediate use in the current terminal.

### 3. Windows user-guide E2E now covers bare Git Bash execution

- The Windows User Guide E2E workflow now installs the package, injects a stale
  old `Scripts\ai-sdlc` directory into PATH, starts Git Bash, and runs
  `ai-sdlc --version`.
- The workflow fails if Git Bash resolves the stale entry or if the reported
  version does not match the release tag.

## Compatibility Notes

- No frontend provider, Vue3 PrimeVue, Vue2 enterprise, or managed delivery
  contract changes are introduced in this patch.
- `ai-sdlc --version` remains the supported success check.
- `python -m ai_sdlc ...` remains available as the module-entry fallback when a
  shell has not refreshed PATH yet.
- Ordinary users should install with `-AddToPath` on Windows, then open a new
  terminal before relying on bare `ai-sdlc ...`.

## Install And Upgrade

Use the platform release asset from
`https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/download/v0.8.10`.

- Windows: download `ai-sdlc-offline-0.8.10-windows-amd64.zip`, then run
  `install_offline.ps1 -AddToPath` for new installs or
  `install_offline.ps1 -UpgradeExisting` when replacing an older CLI.
- macOS / Linux: download the matching `.tar.gz` asset, unpack it, then run
  `install_offline.sh --add-to-path` for new installs or
  `--upgrade-existing` when replacing an older CLI.
- Existing users can try `ai-sdlc self-update check` first. If the output does
  not finish with version `0.8.10`, use the matching platform package and
  `--upgrade-existing`.
- If an older CLI reports `No such command 'install'`, skip diagnosis and use
  the matching platform package with `--upgrade-existing`.
