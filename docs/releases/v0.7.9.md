# AI-SDLC v0.7.9 Release Notes

## Positioning

`0.7.9` is an installed-runtime self-update patch on top of `0.7.8`.

It closes the production gap reported from the `0.7.7` upgrade path: after an offline package upgrade, users were told that future updates only needed `ai-sdlc self-update check`, but a slow or unreachable GitHub latest-release request could leave the command at a generic `Update check failed` notice without applying the update or giving a concrete rescue command.

The intended behavior is now:

- explicit `ai-sdlc self-update check` is treated as a user-requested update attempt, not as a lightweight background notice
- failed update checks do not block the next explicit check through cached backoff
- self-update commands do not mutate project adapter files or pollute JSON output with adapter hook messages
- if latest-release truth still cannot be refreshed, the CLI gives the offline `--upgrade-existing` rescue path

## What Changed

### 1. Explicit self-update check retries with a real user-command timeout

`self-update check` now calls update evaluation with a longer timeout intended for explicit user commands. The shorter update-advisor timeout remains appropriate for background interactive notices.

### 2. Failed-check backoff no longer blocks explicit checks

Background update notices still use failure backoff to avoid noisy repeated network calls. Explicit `self-update check` bypasses that backoff because the user has intentionally asked the CLI to try again.

### 3. Failure output now gives a concrete rescue path

If latest-release truth cannot be refreshed, `self-update check` exits non-zero with a result-oriented panel. It tells the user that the install was not changed, suggests rerunning `ai-sdlc self-update check` after network recovery, and gives the platform offline-package upgrade commands:

- macOS / Linux: `./install_offline.sh --upgrade-existing`
- Windows: `powershell -ExecutionPolicy Bypass -File .\install_offline.ps1 -UpgradeExisting`

Legacy installs that report `No such command 'install'`, or installed runtimes that cannot refresh update state, should download the matching package from `releases/download/v0.7.9` and run `--upgrade-existing` once. After that, return to `ai-sdlc self-update check` for future updates.

### 4. Self-update commands no longer trigger project adapter hooks

`self-update` is now treated as a read-only CLI surface for project adapter purposes. Commands such as `self-update identity --json` no longer apply IDE adapter files or prepend `IDE adapter ...` messages to machine-readable output.

## Compatibility Notes

- macOS / Linux use `.tar.gz` platform packages:
  - macOS Apple Silicon uses `ai-sdlc-offline-0.7.9-macos-arm64.tar.gz`
  - Linux x64 uses `ai-sdlc-offline-0.7.9-linux-amd64.tar.gz`
- Windows x64 uses `ai-sdlc-offline-0.7.9-windows-amd64.zip`
- Users already stuck on `0.7.7` or `0.7.8` with `Update check failed` should download the matching `0.7.9` offline package and run `--upgrade-existing` once.
- After `0.7.9` is installed, normal future updates should return to `ai-sdlc self-update check`.
