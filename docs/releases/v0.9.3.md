# AI-SDLC v0.9.3 Release Notes

## Positioning

`0.9.3` is a Vue3 frontend governance release. It upgrades the
`public-primevue` default provider line to the Vue3 enterprise frontend standard
v1.3 AI-executable contract.

This is not a provider switch. The ordinary default remains
`frontend_stack=vue3`, `provider_id=public-primevue`, and
`style_pack_id=modern-saas`. The release tightens the standard around how AI
agents should generate Vue3 enterprise UI once the user has confirmed that
frontend solution.

## What Changed

### 1. Vue3 enterprise standard v1.3 is now the canonical frontend spec

- `docs/Vue3企业级前端开发规范方案.md` is now the v1.3 AI-executable standard.
- The spec keeps the default stack at Vue3, TypeScript, Vite, PrimeVue,
  `@primeuix/themes`, `primeicons`, UnoCSS, CSS Variables, Pinia, Vue Router,
  Axios, `vee-validate`, `zod`, and `vue-i18n`.
- It adds explicit visual-family rules for navigation, page headers, badges,
  buttons, tags, filters, and form controls.
- It documents the difference from the older standard so downstream AI agents
  can see what changed and why.

### 2. The provider and generation governance now consume the same rules

- `public-primevue` template development dependencies now include
  `husky`, `lint-staged`, `@commitlint/cli`, and
  `@commitlint/config-conventional`.
- Generation hard rules now cover visual-family consistency, native form
  control mixing, development-time `$i('中文')` visible-copy handling, and
  semantic tag severity mapping.
- Token rules now make native select fallback, raw visible Chinese without an
  i18n wrapper, and page-local `severity=contrast` shortcuts observable
  generation risks.

### 3. CLI and adapter guidance are aligned with the v1.3 standard

- `program solution-confirm` now presents the default Vue3 tooling as:
  `Vitest + Playwright + ESLint + Prettier + husky + lint-staged + commitlint`.
- AGENTS / Codex / Claude Code / VS Code / Cursor adapter templates use the
  same frontend confirmation wording.
- `verify constraints` checks the same token set, so future adapter drift is
  caught as a release blocker.

### 4. Validation remains staged instead of over-blocking ordinary generation

- `husky`, `lint-staged`, and `commitlint` are part of the default engineering
  recommendation and dependency truth, but missing full project wiring remains
  advisory for ordinary generated frontends.
- `$i('中文')`, visual-family consistency, native form-control mixing, and
  `contrast` tag misuse are now explicit governance risks. They can be reported
  before being upgraded into hard blockers in a later release.

## Compatibility Notes

- This release does not remove the Vue2 compatibility provider or the advanced
  frontend solution-selection path.
- Vue3 `public-primevue` with `style_pack_id=modern-saas` remains the ordinary
  first recommendation for new frontend work.
- Explicit `vue2` / `enterprise-vue2` requests must still be honored.
- `ai-sdlc init .` remains the beginner setup entry. `adapter status` and
  `run --dry-run` remain troubleshooting commands, not required beginner-path
  setup steps.
- `python -m ai_sdlc ...` remains available as the module-entry fallback when a
  shell has not refreshed PATH yet.

## Install And Upgrade

Use the platform release asset from
`https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/download/v0.9.3`.

- Windows: download `ai-sdlc-offline-0.9.3-windows-amd64.zip`, then run
  `install_offline.ps1 -AddToPath` for new installs or
  `install_offline.ps1 -UpgradeExisting` when replacing an older CLI.
- macOS / Linux: download the matching `.tar.gz` asset, unpack it, then run
  `install_offline.sh --add-to-path` for new installs or
  `--upgrade-existing` when replacing an older CLI.
- Existing users can try `ai-sdlc self-update check` first. If the output does
  not finish with version `0.9.3`, use the matching platform package and
  `--upgrade-existing`.
- If an older CLI reports `No such command 'install'`, skip diagnosis and use
  the matching platform package with `--upgrade-existing`.
