# AI-SDLC v0.9.6 Release Notes

## Positioning

`0.9.6` is the Vue3 visual-contract refresh release for the
`public-primevue` default provider line. It upgrades the default Vue3 enterprise
standard from v1.7.1 to the v1.8 AI-executable contract and carries the new
light-page information-carrier consistency rule into provider truth,
generation governance, adapter guidance, and release validation.

This is not a provider switch. The ordinary default remains
`frontend_stack=vue3`, `provider_id=public-primevue`, and
`style_pack_id=modern-saas`. The release tightens the visual evidence contract:
ordinary cards, panels, filters, stat blocks, tag groups, and information
groups in light pages must remain in the same visual family as the page body.

## What Changed

### 1. Vue3 enterprise standard v1.8 is now canonical

- `docs/Vue3企业级前端开发规范方案.md` now carries the v1.8 AI-executable
  visual standard.
- The v1.8 contract inherits the v1.7.1 path-level structure, theme-entry,
  API, DTO transform, route meta, i18n, and normative/advisory/landed-fact
  boundaries.
- The new hard rule is `light-surface-information-carrier-consistency`; the
  warning evidence token is `ordinary-carrier-visual-family-drift`.

### 2. Public PrimeVue provider truth is versioned to v1.8

- `providers/frontend/public-primevue/provider.manifest.yaml` now records
  `visual_contract_version: v1.8`.
- `verify constraints` blocks stale `public-primevue` provider manifests that
  still declare the old visual contract.
- Generation hard rules and token warning rules now include the light-page
  information-carrier consistency contract.

### 3. Agent and adapter guidance is aligned

- AGENTS / Codex / Claude Code / VS Code / Cursor guidance now repeats the
  v1.8 light-page information-carrier rule beside the existing
  `primary / surface / highlight` and `theme.ts` single-entry requirements.
- The rule stays scoped to frontend work and does not change the
  `vue2` / `enterprise-vue2` explicit compatibility path.

## Compatibility Notes

- Existing Vue2 compatibility and advanced frontend solution selection remain
  unchanged.
- Vue3 `public-primevue` with `style_pack_id=modern-saas` remains the ordinary
  first recommendation for new frontend work.
- Existing projects using `src/views` continue to be scanned by the import
  boundary checks.
- `ai-sdlc init .` remains the beginner setup entry. `adapter status` and
  `run --dry-run` remain troubleshooting commands, not required beginner-path
  setup steps.
- `python -m ai_sdlc ...` remains available as the module-entry fallback when a
  shell has not refreshed PATH yet.

## Install And Upgrade

Use the platform release asset from
`https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/download/v0.9.6`.

- Windows: download `ai-sdlc-offline-0.9.6-windows-amd64.zip`, then run
  `install_offline.ps1 -AddToPath` for new installs or
  `install_offline.ps1 -UpgradeExisting` when replacing an older CLI.
- macOS / Linux: download the matching `.tar.gz` asset, unpack it, then run
  `install_offline.sh --add-to-path` for new installs or
  `--upgrade-existing` when replacing an older CLI.
- Existing users can try `ai-sdlc self-update check` first. If the output does
  not finish with version `0.9.6`, use the matching platform package and
  `--upgrade-existing`.
- If an older CLI reports `No such command 'install'`, skip diagnosis and use
  the matching platform package with `--upgrade-existing`.
