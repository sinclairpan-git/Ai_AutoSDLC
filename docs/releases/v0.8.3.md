# AI-SDLC v0.8.3 Release Notes

## Positioning

`0.8.3` is a frontend-governance safety patch. It keeps the user-facing install
and upgrade commands unchanged while making frontend/UI delivery stop for an
explicit technology-stack and component-library confirmation before any
implementation work begins.

## What Changed

### 1. Frontend solution confirmation is required before implementation

- Frontend, UI, page, component, browser-interaction, and frontend-engineering
  requirements must now present technology stack and component-library
  recommendations before `execute`.
- The agent must wait for explicit user confirmation before writing frontend
  implementation code, entering `execute`, or running managed delivery apply.
- Legal next steps before confirmation are refine, design, decompose, or a
  confirmation card.

### 2. Built-in Vue2 enterprise provider is protected

- When the user's target framework is the built-in Vue2 enterprise component
  library, the default recommendation must include `enterprise-vue2` / `vue2`.
- The rules now explicitly forbid silently switching to `public-primevue`, Vue3,
  React, or another public fallback when the user expects the built-in Vue2
  enterprise path.
- The confirmed solution record must capture `frontend_stack`, `provider_id`,
  component library/source, style scheme, and whether the built-in provider is
  used.

### 3. Release constraints guard the instruction surfaces

- `verify constraints` now checks the pipeline rules, AGENTS guidance, and
  Codex / Claude Code / VS Code / Cursor adapter templates for the frontend
  confirmation markers.
- The release entry docs, workflow defaults, and Windows upgrade smoke expected
  version are aligned to `v0.8.3`.
- The framework defect backlog records the 2026-06-10 frontend-flow breach for
  future regression tracking.

## Compatibility Notes

- Windows x64 uses `ai-sdlc-offline-0.8.3-windows-amd64.zip`.
- macOS / Linux use `.tar.gz` bundles:
  `ai-sdlc-offline-0.8.3-macos-arm64.tar.gz` and
  `ai-sdlc-offline-0.8.3-linux-amd64.tar.gz`.
- Existing `v0.7.x` and `v0.8.x` users can upgrade directly to `v0.8.3` with the
  matching platform package or `ai-sdlc self-update check`.
- Users stuck on older releases, including CLIs that report
  `No such command 'install'`, should download the matching package from
  `releases/download/v0.8.3` and run `--upgrade-existing` once from the unpacked
  bundle directory.
- Users who want direct `ai-sdlc ...` commands should install with `-AddToPath`
  on Windows or `--add-to-path` on macOS / Linux.
- Users who cannot rely on `PATH` can keep using `python -m ai_sdlc ...` from
  the installed runtime.

- `uv run pytest`

- `uv run ai-sdlc verify constraints`
- GitHub PR checks for this release branch

- `release-build.yml` for tag `v0.8.3`
- `release-artifact-smoke.yml` for tag `v0.8.3`
