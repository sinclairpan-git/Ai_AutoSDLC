# AI-SDLC v0.8.6 Release Notes

## Positioning

`0.8.6` is a post-`0.8.5` user-experience patch for Windows upgrade and
first-run guidance. It keeps the Vue3 `public-primevue` default-provider
behavior and focuses on removing internal adapter diagnostics from ordinary
user-facing output.

## What Changed

### 1. Windows `project-config.yaml` metadata locks no longer block dry-run

- If Windows or the AI host temporarily locks
  `.ai-sdlc/project/config/project-config.yaml`, the automatic adapter hook now
  treats that as a non-blocking metadata-write warning.
- The warning is narrowly scoped to `project-config.yaml`; permission failures
  while creating or updating canonical adapter files such as `AGENTS.md` still
  surface as real setup problems.
- `ai-sdlc run --dry-run` can continue so users do not mistake a transient
  metadata lock for a frontend implementation or code-generation failure.

### 2. Ordinary output hides internal adapter proof terminology

- Default `adapter status`, `run --dry-run`, and `host-runtime plan` output no
  longer exposes internal wording such as `ingress`, `materialized`,
  `unverified`, or governance activation proof.
- The normal path now tells users that project rules are ready and that they
  should continue in the AI chat or run the safe rehearsal only for
  troubleshooting.
- Raw diagnostic state remains available through explicit troubleshooting
  surfaces such as `--json` and `--details`.

### 3. Update advisor wording is clearer for installed users

- The update advisor no longer says the install channel is "not confirmed for
  automatic upgrade" in the ordinary fallback case.
- When the current CLI entry cannot be safely replaced in place, the message now
  says to use the offline installer or the company/project-provided install
  entry.
- Non-required AgentOps pending diagnostics stay quiet unless verbose output is
  enabled.

## Compatibility Notes

- No provider, scaffold, workflow, or frontend governance contract changes are
  introduced in this patch.
- Existing projects that explicitly selected `enterprise-vue2` or already use
  the Vue3 public PrimeVue default continue to behave the same.
- Machine-readable adapter diagnostics keep their raw state names for framework
  and CI consumers.

## Install And Upgrade

Use the platform release asset from
`https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/download/v0.8.6`.

- Windows: download the `.zip` asset, for example
  `ai-sdlc-offline-0.8.6-windows-amd64.zip`, then run
  `install_offline.ps1 -AddToPath` for new installs or
  `install_offline.ps1 -UpgradeExisting` when replacing an older CLI.
- macOS / Linux: download the matching `.tar.gz` asset, unpack it, then run
  `install_offline.sh --add-to-path` for new installs or
  `--upgrade-existing` when replacing an older CLI.
- If an older installed CLI reports `No such command 'install'`, replace it
  once with the matching platform asset and `--upgrade-existing`; after that,
  use `ai-sdlc self-update check` for future releases.
- If PATH has not been refreshed yet, use the bundle Python entry directly with
  `python -m ai_sdlc`.

Release assets are produced by `release-build.yml`. After the GitHub Release is
published and assets are attached, `release-artifact-smoke.yml` verifies the
published `.zip` and `.tar.gz` assets from the release page.
