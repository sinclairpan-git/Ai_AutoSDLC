# AI-SDLC v0.8.2 Release Notes

## Positioning

`0.8.2` is an init and upgrade-path reliability patch. It keeps the user-facing
install and upgrade commands unchanged while making successful `init` runs easier
to recognize from later stages and making old-entrypoint upgrade guidance more
truthful.

## What Changed

### 1. Init completion now depends on the init gate

- `ai-sdlc init .` records `init` as completed only after the init gate passes.
- A damaged initialized project no longer keeps a stale completed `init` ledger
  entry when required init evidence is missing.
- `ai-sdlc run --dry-run` can continue from `refine` after a valid init instead
  of replaying `init` just because the checkpoint still names the init stage.

### 2. Upgrade guidance reports the entrypoint that is actually running

- Upgrade diagnostics distinguish the source checkout from the installed
  `ai-sdlc` entry currently resolved by the shell.
- Existing users can still run `ai-sdlc self-update check` when that command is
  available.
- Users stuck on older releases, including CLIs that report
  `No such command 'install'`, should download the matching package from
  `releases/download/v0.8.2` and run `--upgrade-existing` once from the unpacked
  bundle directory.

### 3. AgentOps and verification summaries avoid misleading payloads

- Passing gate facts no longer include a failed-command field.
- Candidate fix lists are summarized instead of emitted as raw path-heavy lists.
- Verification metrics count public `test` stage checks instead of mixing in
  hidden/internal checks.

## Compatibility Notes

- Windows x64 uses `ai-sdlc-offline-0.8.2-windows-amd64.zip`.
- macOS / Linux use `.tar.gz` bundles:
  `ai-sdlc-offline-0.8.2-macos-arm64.tar.gz` and
  `ai-sdlc-offline-0.8.2-linux-amd64.tar.gz`.
- Existing `v0.7.x` and `v0.8.x` users can upgrade directly to `v0.8.2` with the
  matching platform package or `ai-sdlc self-update check`.
- Users who want direct `ai-sdlc ...` commands should install with `-AddToPath`
  on Windows or `--add-to-path` on macOS / Linux.
- Users who cannot rely on `PATH` can keep using `python -m ai_sdlc ...` from
  the installed runtime.

## Verification

Required before marking `v0.8.2` complete:

- `uv run pytest`
- `uv run ruff check src tests`
- `uv run ai-sdlc verify constraints`
- GitHub PR checks for this release branch
- Codex review with no unresolved blocking findings
- `release-build.yml` for tag `v0.8.2`
- `release-artifact-smoke.yml` for tag `v0.8.2`
