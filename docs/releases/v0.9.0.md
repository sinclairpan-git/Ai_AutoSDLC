# AI-SDLC v0.9.0 Release Notes

## Positioning

`0.9.0` is a frontend solution-governance release. It fixes the regression where
new frontend requirements could still receive `enterprise-vue2` / Vue2 as the
ordinary first recommendation after the Vue3 provider policy had already been
approved.

The release makes Vue3 `public-primevue` the default recommendation path,
surfaces the actual component library and tooling in beginner output, and
restores advanced multi-solution choice so experienced users can still select a
different stack or style deliberately.

## What Changed

### 1. Vue3 PrimeVue is the default frontend provider

- The default frontend solution confirmation path now recommends
  `frontend_stack=vue3`, `provider_id=public-primevue`, and
  `style_pack_id=modern-saas`.
- Simple confirmation output now shows `PrimeVue + @primeuix/themes` and
  `Vite + TypeScript + UnoCSS + CSS Variables`, not only internal provider IDs.
- Enterprise Vue2 remains available only as an explicit compatibility choice,
  not the ordinary default for new frontend work.

### 2. Advanced frontend choices are restored

- Advanced mode keeps Vue3 style alternatives such as `modern-saas`,
  `data-console`, `high-clarity`, and `macos-glass`.
- `--provider-id` is now part of the guarded confirmation surface, so provider
  choice cannot silently disappear from CLI guidance or adapter rules.
- Candidate command hints preserve preflight and fallback context, including
  ineligible enterprise-provider states and no-fallback states.

### 3. Regression guards now protect old and new behavior together

- Release constraints now check the Vue3 default recommendation, advanced
  choice tokens, beginner output details, and the existing enterprise Vue2
  compatibility path.
- Feature-regression review rules now require old user-visible capabilities to
  stay covered unless a breaking change is explicitly documented.
- Verification-profile and feature-regression guards are scoped to framework
  rule surfaces so ordinary user projects are not blocked by framework-only
  checks.

### 4. Windows install and upgrade hardening remains current

- The `0.8.10` stable Windows command shim behavior remains in this release.
- Windows installers continue to prioritize `%LOCALAPPDATA%\AI-SDLC\bin` for
  new PowerShell, cmd, and Git Bash sessions.
- `ai-sdlc --version` remains the supported success check.

## Compatibility Notes

- This is a minor release because it changes the ordinary recommendation policy
  for frontend solution confirmation.
- Existing projects that explicitly choose `enterprise-vue2` / Vue2 remain
  supported.
- `ai-sdlc init .` remains the ordinary setup entry. `adapter status` and
  `run --dry-run` remain troubleshooting commands, not beginner-path setup
  steps.
- `python -m ai_sdlc ...` remains available as the module-entry fallback when a
  shell has not refreshed PATH yet.

## Install And Upgrade

Use the platform release asset from
`https://github.com/sinclairpan-git/Ai_AutoSDLC/releases/download/v0.9.0`.

- Windows: download `ai-sdlc-offline-0.9.0-windows-amd64.zip`, then run
  `install_offline.ps1 -AddToPath` for new installs or
  `install_offline.ps1 -UpgradeExisting` when replacing an older CLI.
- macOS / Linux: download the matching `.tar.gz` asset, unpack it, then run
  `install_offline.sh --add-to-path` for new installs or
  `--upgrade-existing` when replacing an older CLI.
- Existing users can try `ai-sdlc self-update check` first. If the output does
  not finish with version `0.9.0`, use the matching platform package and
  `--upgrade-existing`.
- If an older CLI reports `No such command 'install'`, skip diagnosis and use
  the matching platform package with `--upgrade-existing`.
